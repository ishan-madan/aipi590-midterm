# Sleep Comfort AI (Raspberry Pi)

## Overview
This project is a real-time AI-powered sleep comfort monitoring system deployed on a Raspberry Pi. It uses temperature, humidity, and motion sensors to predict sleep quality, displaying results on an LCD and controlling LEDs. All readings are logged for later analysis.  

---

## Features
- Reads temperature & humidity from a **DHT11** sensor.  
- Reads motion magnitude from an **MPU6050** accelerometer.  
- Logs all readings and AI predictions to a CSV file.  
- Loads a trained **Logistic Regression model** (`sleep_model.pkl`) to predict sleep quality probability.  
- Displays current sleep comfort on a **16x2 LCD**.  
- Controls **Green and Red LEDs** based on prediction.  
- Waits for a **physical button press** to start monitoring.  

---

## Hardware Required
- Raspberry Pi (any model with GPIO & I2C support)  
- **DHT11** temperature/humidity sensor (GPIO 16)  
- **MPU6050** accelerometer (I2C)  
- **16x2 LCD** with I2C interface (address 39)  
- **Green LED** (GPIO 21)  
- **Red LED** (GPIO 20)  
- **Push Button** (GPIO 5)  

---

## Wiring
- **Button:** GPIO 5 → Button → GND (use `pull_up=True` in code)  
- **DHT11:** GPIO 16 → DHT11 Data  
- **MPU6050:** I2C SDA/SCL → Pi SDA/SCL  
- **LEDs:** GPIO 21 (Green), GPIO 20 (Red) → appropriate resistors → GND  
- **LCD:** I2C SDA/SCL → Pi SDA/SCL

## Code Structure
The code is organized in a single Python script `sleep_comfort_ai.py` with the following structure:

1. **Imports and Constants**  
   - Imports hardware, AI, and utility libraries.  
   - Defines GPIO pins, LCD size, thresholds, and log file paths.

2. **Sensor Initialization Functions**
   - `initialize_dht()`: Sets up DHT11 sensor.  
   - `initialize_mpu()`: Sets up MPU6050 accelerometer.  
   - `initialize_lcd()`: Sets up 16x2 LCD display.  
   - `initialize_leds()`: Configures Green and Red LEDs.

3. **Sensor Reading Functions**
   - `read_dht(sensor)`: Returns temperature and humidity.  
   - `read_motion(mpu)`: Returns motion magnitude from accelerometer.

4. **Logging Functions**
   - `initialize_log()`: Creates CSV file with headers if missing.  
   - `log_data(temp, humidity, motion, probability)`: Appends one row to the CSV log.

5. **Model Integration**
   - `load_model()`: Loads trained Logistic Regression model (`sleep_model.pkl`).  
   - `predict_sleep_quality(model, temp, humidity, motion)`: Returns probability of good sleep.

6. **Output Functions**
   - `update_outputs(probability, lcd, green_led, red_led)`: Updates LCD and LEDs based on sleep probability.

7. **Main Loop (`main()`)**
   - Initializes sensors, LEDs, LCD, log file, and button.  
   - Displays `"Click button to start"` on LCD.  
   - Waits for the user to press the physical start button.  
   - Continuously reads sensors, predicts sleep quality, logs data, and updates outputs.  
   - Stops safely on `KeyboardInterrupt`.

## Installation
1. Clone the repository.  
2. Install dependencies:  
   pip install adafruit-circuitpython-dht adafruit-circuitpython-mpu6050 gpiozero joblib
3. Place `sleep_model.pkl` in the project directory.  

---

## Usage
1. Connect hardware according to the wiring diagram.  
2. Run the script:  
   python sleep_comfort_ai.py
3. The LCD will show:  
   Click button to  
   start
4. Press the physical button to begin monitoring.  
5. The system will log data to `sleep_runtime_log.csv`, update LEDs, and display current sleep comfort on the LCD.  

---

## Logging
All runtime readings are saved in `sleep_runtime_log.csv` with the following columns:  
timestamp, temperature, humidity, motion, sleep_probability
